# Court IQ v0.1 Demo Checklist

Use this checklist 15–20 minutes before any live Court IQ v0.1 demo. It is local-first and does not require external video or network services once dependencies are installed.

## 1. Environment setup

- [ ] Open two terminals from repo root (`/workspace/basketball-decisions`).
- [ ] Confirm Node/npm are available for frontend.
- [ ] Confirm Python 3 is available for backend.
- [ ] Confirm ports are free:
  - Backend: `8000`
  - Frontend: `5173`

## 2. Backend start

- [ ] In terminal A, start backend from documented quick-start:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- [ ] Wait for server-ready log line and no startup errors.
- [ ] Health check:

```bash
curl http://localhost:8000/api/sample-data/status
```

## 3. Frontend start

- [ ] In terminal B, start frontend from documented quick-start:

```bash
cd frontend
npm install
npm run dev
```

- [ ] Open `http://localhost:5173`.
- [ ] Confirm initial route loads and navigation menu is visible.

## 4. Seed sample data

- [ ] Check existing sample-data status:

```bash
curl http://localhost:8000/api/sample-data/status
```

- [ ] Seed deterministic sample project:

```bash
curl -X POST http://localhost:8000/api/sample-data/seed
```

- [ ] Re-check seeded status:

```bash
curl http://localhost:8000/api/sample-data/status
```

## 5. Verify Development Dashboard

- [ ] Visit `http://localhost:5173/development-dashboard`.
- [ ] Confirm dashboard loads without empty/error state.
- [ ] Confirm workflow entry points are clickable.

## 6. Verify Player Home

- [ ] Visit `http://localhost:5173/player-home`.
- [ ] Confirm player card/home summary renders.
- [ ] Confirm no blocking API errors appear.

## 7. Verify Player Value

- [ ] Visit `http://localhost:5173/player-value`.
- [ ] Confirm value summary metrics render.
- [ ] Confirm trends/detail links open without crash.

## 8. Verify Coach Report Summary

- [ ] Visit `http://localhost:5173/coach-reports`.
- [ ] Confirm report summary sections populate.
- [ ] Confirm recommendation content is readable.

## 9. Verify Drills

- [ ] Visit `http://localhost:5173/drills`.
- [ ] Confirm drill catalog entries render.
- [ ] Confirm at least one drill detail opens.

## 10. Verify Practice Plan

- [ ] Visit `http://localhost:5173/practice-plans`.
- [ ] Confirm practice plan list and/or generated plan appears.
- [ ] Confirm plan blocks can be reviewed in sequence.

## 11. Verify Practice Execution

- [ ] Visit `http://localhost:5173/practice-executions`.
- [ ] Confirm execution list/table renders.
- [ ] Open one execution detail and confirm no blocking errors.

## 12. Verify Workflow

- [ ] Visit `http://localhost:5173/workflows`.
- [ ] Confirm workflow states and detail navigation work.
- [ ] Open one workflow detail page and verify status content.

## 13. Run smoke test

- [ ] From repo root, run documented smoke/e2e suite:

```bash
npm run test:e2e
```

- [ ] Confirm test run passes (or capture known deterministic failure before demo).

## 14. Known caveats to say out loud

- [ ] "This is a v0.1 local-first demo with deterministic sample data."
- [ ] "Outputs are designed for explainability and workflow validation, not live game inference."
- [ ] "Data is seeded from local fixtures; we are not using external video/network feeds in this demo."
- [ ] "Some pages may show placeholder/fallback content where production integrations are intentionally deferred."

## 15. Demo recovery steps

- [ ] If frontend appears stale, hard refresh browser tab.
- [ ] If sample data appears missing, reseed:

```bash
curl -X POST http://localhost:8000/api/sample-data/seed
```

- [ ] If backend is inconsistent, restart backend process and repeat seed step.
- [ ] If frontend cannot reach API, confirm backend still running on `http://localhost:8000` and frontend on `http://localhost:5173`.
- [ ] If demo flow breaks, return to `http://localhost:5173/development-dashboard` and restart narrative from command-center view.
