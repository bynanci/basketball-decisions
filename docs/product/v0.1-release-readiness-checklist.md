## Status legend

- **PASS**: Requirement is complete and verified with evidence.
- **FAIL**: Requirement is not met; release cannot proceed until fixed or formally removed from scope.
- **PARTIAL**: Requirement is partially met; explicit exception and approver are required for GO.
- **NOT APPLICABLE**: Item does not apply to v0.1 scope and includes rationale in Notes.

## How to use this checklist

- Every item must include a **Status**, **Owner**, and **Notes**.
- Notes should reference evidence (test command, doc link, or rehearsal date).
- If Status is **PARTIAL** or **FAIL**, Notes must include next action and target date.

## v0.1 GO / NO-GO criteria (all must be PASS)

Release can be tagged/demoed as **v0.1 GO** only when **all** criteria below are **PASS**.  
If any criterion is **FAIL**, decision is **NO-GO**.  
If any criterion is **PARTIAL**, release remains **NO-GO** until a written exception is approved by Product + Engineering leads and logged at the bottom of this document.
